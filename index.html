HTML

  • The body element represents the visible content shown to the user.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status:this checks what branch we are currently on and changes we have made
  • git pull origin main:pulls down the latest main branch (i.e., gets the latest copy)
  • git checkout -b feature/branch-name: creates a new branch and switches to it (or you replace "feature" with "develop")
  • STEP 1: git add -A: commits all files in the working branch, also called staging
  • STEP 2: git commit -m "added code to HTML file": commits staging to our repository (the languaged quoted is the title of the pull request in github
  • BEST PRACTICE: git pull origin main: makes sure that our current branch is up to date before opening a pull request; you want the message "Already up to date"
  • STEP 3: git push origin feature/branch-name: pushes up all of your changes
  • then go to github repository and you should see a pending pull request
